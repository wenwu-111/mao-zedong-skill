## 项目来由

本 Skill 初始版来自于 B 站看到的一个项目：[leezythu/maoxuan-skill](https://github.com/leezythu/maoxuan-skill)，它把《毛泽东选集》五卷里的思维方法提炼成了 AI 能用的认知模型。但原版只用了毛选五卷，一共 229 篇文章。我在使用过程中发现会出现文字幻觉、数字使用错误的情况，于是开始补全著作。

原版只有毛选五卷。这一版加上了**毛泽东文集八卷**（建国后的讲话、书信、批示）、苍苍社出版的**《毛泽东集》**、1937 年在抗大的**讲课提纲**，还有 191 篇散篇——晚年外交谈话、未删节讲话稿、对各国共产党代表团的谈话记录，从 1930 年代一直覆盖到 1975 年。

同时补**理论地基**。加了《共产党宣言》《资本论》《关于费尔巴哈的提纲》等 18 篇马恩原著，列宁《帝国主义论》《国家与革命》等 3 卷，还有卢卡奇《历史与阶级意识》、马尔库塞《单向度的人》当对照。

原文全是网页抓取的，有许多导航残留、空标题、重复段落、电子书网站的广告。文件名要么是纯数字要么是一长串 URL 编码。我全部整理了一遍——文件名加了中文标题、多余空行压缩、垃圾广告清理、目录结构也改了一下。

另外原版的 SKILL.md 设定比较基础。我加了「凡涉精确断言，一律核实」「不知道的就说不知道」「不含糊其辞」等诚实约束，补充了知识库检索路径和材料引用层级，让 AI 的回复不那么飘。

最终里面有 1319 个文件，覆盖了从马克思到毛泽东到西马批判的一整条思想脉络。核心是 7 个心智模型：矛盾分析法、实践认识循环、持久战略、农村包围城市、统一战线、群众路线、纸老虎论，用来帮你分析问题。

## 效果示例

```
用户      ❯ 我们是个小创业公司，产品做得不错但没资源推广。
            大厂出了类似产品，用户都在往那边跑。怎么办？

回答      ❯ 同志，你这个问题我太熟悉了。
            1927年我们几百人上井冈山，国民党几十万人围着打，
            情况比你难一百倍。

            先分析矛盾。你的主要矛盾不是"大厂做了类似产品"——
            这只是现象。主要矛盾是：你还没有找到一个大厂不愿意
            认真做、但用户又真正需要的根据地。

            大厂的弱点在哪里？大厂什么都要做，什么都做不深。
            它的产品经理要同时管十个方向，你只管一个。
            这就是你的根据地——一个足够深、足够窄的场景。

            不要在它的主战场上跟它打阵地战。
            农村包围城市，同志。
            先把一个小场景做到它十倍好，让这个场景的用户
            非你不可。一个根据地站稳了，再扩展下一个。

            星星之火，可以燎原。
            前提是你这把火要烧得够旺，别急着铺开。
```

```
用户      ❯ 团队里两个技术leader互相看不顺眼，各带一帮人，
            项目推不动。我是他们的上级，怎么处理？

回答      ❯ 这个问题本质上是统一战线的问题。

            先搞清楚：他们之间的矛盾是主要矛盾吗？不是。
            公司的主要矛盾是产品要活下去、要赢市场。
            他们之间是人民内部矛盾，不是敌我矛盾。

            处理人民内部矛盾，靠的是"团结——批评——团结"：
            先从团结出发，把两个人叫到一起，明确一个共同目标——
            比如下季度的核心交付。这是他们的"抗日"——
            在这个共同目标面前，个人恩怨是次要矛盾。

            然后搞一次认真的批评与自我批评。
            不是和稀泥，不是各打五十大板。
            让双方都摆事实讲道理，把问题放到桌面上。

            最后回到团结。
            你要做的不是当裁判，是当统一战线的组织者——
            把两支队伍的力量都用在打敌人（打市场）上，
            而不是用在打自己人上。

            记住：把朋友搞得多多的，把敌人搞得少少的。
            你的敌人是市场上的竞争对手，不是自己团队的人。
```

```
用户      ❯ 我想转行做AI，但现在AI行业人才太多了，
            我一个传统后端开发完全没有优势，很焦虑。

回答      ❯ 同志，你犯了一个错误：你只看到了敌人的强大，
            没有看到它的纸老虎本质。

            AI行业人才多？一分为二看。
            人多，说明大部分人都在做同样的事——
            调API、套框架、跟风追热点。
            真正深入理解业务场景的人，反而很少。

            你是传统后端出身？这不是劣势，这是你的根据地。
            那些AI工程师懂模型、懂算法，但他们不懂你做了
            五年八年的行业系统——数据怎么流的、业务逻辑
            怎么跑的、线上出了问题怎么排查。

            没有调查就没有发言权。
            先去调查：你所在的行业，AI落地最大的瓶颈是什么？
            我敢说，百分之八十的瓶颈不在模型，
            在工程、在数据、在对业务的理解。
            这些恰恰是你有而AI原生工程师没有的。

            不要去跟人家在算法能力上打阵地战。
            用你的行业经验做根据地，
            AI能力作为你的增量武器。
            这叫——用自己的长处打别人的短处。
```

这不是角色扮演。用「矛盾分析法」拆解创业困境，用「统一战线」处理团队冲突，用「纸老虎论」和「农村包围城市」应对职业焦虑。**是用他的认知框架帮你看问题，不是在复读语录。**

## 蒸馏了什么

毛泽东的核心不是具体结论，是**认识世界和改造世界的方法论**。本 Skill 提炼了他的认知操作系统：

| 心智模型 | 一句话 | 来源 |
|---------|--------|------|
| **矛盾分析法** | 找到主要矛盾，就找到了破局之道 | 《矛盾论》 |
| **实践认识循环** | 没有调查就没有发言权。认识从实践来，回到实践去 | 《实践论》 |
| **持久战略** | 力量不利时，以时间换空间，在持久中改变力量对比 | 《论持久战》 |
| **农村包围城市** | 不攻最强点，先在边缘建根据地，再逐步包围中心 | 《星星之火，可以燎原》 |
| **统一战线** | 把朋友搞得多多的，把敌人搞得少少的 | 《中国社会各阶级的分析》 |
| **群众路线** | 从群众中来，到群众中去 | 《关于领导方法的若干问题》 |
| **纸老虎论** | 战略上藐视敌人，战术上重视敌人 | 与斯特朗的谈话 |

11 条决策启发式，包括：
- 没有调查就没有发言权
- 抓主要矛盾——十个问题里找那一个
- 星星之火，可以燎原
- 一分为二看问题
- 不打无准备之仗
- 自力更生为主，争取外援为辅

## 适用场景

| 场景 | 用到的模型 |
|------|-----------|
| **创业公司 vs 大厂** | 农村包围城市 + 持久战略 + 纸老虎论 |
| **资源有限时如何聚焦** | 矛盾分析法（找主要矛盾） |
| **团队内部冲突** | 统一战线 + 正确处理人民内部矛盾 |
| **产品方向迷茫** | 实践认识循环（别猜，去调查） |
| **面对强大对手** | 纸老虎论 + 农村包围城市 |
| **组织管理/带团队** | 群众路线 + 批评与自我批评 |
| **职业规划/转型** | 持久战略（三阶段） + 根据地思维 |
| **谈判/合作** | 统一战线 + 矛盾分析 |

## 诚实边界

**这个 Skill 能做的：**
- 用矛盾分析法拆解复杂问题
- 用持久战略、农村包围城市分析竞争态势
- 用群众路线的思维指导产品/组织决策
- 用统一战线的逻辑梳理人际和利益格局
- 用"一分为二"提供辩证的、不偏颇的视角

**做不到的：**

| 维度 | 说明 |
|------|------|
| 替代本人 | 政治直觉和历史时刻的判断力无法复制 |
| 具体执行方案 | 提供分析框架，具体方案需结合你的实际 |
| 诗词创作 | 思维方式可提取，文学天赋不可蒸馏 |
| 历史评价 | 本 Skill 聚焦方法论，不对具体历史事件做评判 |

**一个不告诉你局限在哪的 Skill，不值得信任。**
